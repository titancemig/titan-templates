Radx CIDD support

Overview

The CIDD display is widely used at NCAR for viewing gridded data, both in Cartesian and radial formats.

CIDD generally reads and displays gridded data from MDV files. See the TITAN documentation for more details on MDV. CIDD can read the data directly from the MDV files. However, in many situations the MDV data is served to CIDD via the DsMdvServer application.

A secondary server application, the NcMdvServer, provides an on-the-fly translation service between netCDF and MDV file formats.

The NcMdvServer has been upgraded to handle all 4 radial data types in common use at NCAR, namely:

  • CfRadial: CF-compliant netCDF (new)
  • Dorade: Doppler Radar Data Exchange Format
  • UF:Universal Format
  • FORAY netCDF: mainly used for TIMREX field experiment

To view these data in CIDD, it is no longer necessary to convert the data to MDV. Instead, the NcMdvServer can serve the data to CIDD on-the-fly.

You will need to ensure that both the DsMdvServer and NcMdvServer are running on the host on which the data is stored. Alternatively, if the DsServerMgr (server manager) is running, it will start up the required servers.

Naming conventions

In order to view radial data directly with CIDD, you need to ensure that the directory structure and file names comply with the required naming conventions.

The directory naming rules are:

  • Each data set should reside in a separate directory
  • The data for each day should reside in a sub-directory named 
        yyyymmdd
  • The file name should contain the date and time - see below

The files should either be named using the standard DORADE file naming convention, or should contain the date and time in a standard format.

  • CfRadial naming convention
  • DORADE naming convention
  • UF naming convention
  • Foray netCDF naming convention

The following show examples of full paths:

  • /data/cfradial/netcdf4/20080604/cfrad.20080604_002217.nc: CfRadial file
  • /data/dorade/timrex/20080604/swp.1080604001951.SPOLRVP8.0.4.7_SUR_v036: DORADE file
  • /data/uf/timrex/20080604/20080604_002217_SPOLRVP8_SUR_v036.uf: UF file
  • /data/foraync/timrex/20080605/ncswp_SPOL_20080605_235100.000_v054_s007_009.1_SUR_.nc: FORAY netCDF file

CfRadial data naming convention

CfRadial files have the following naming convention: 

    *cfrad*yyyymmdd_hhmmss*.nc
where * refers to any character string.

For example, the file: 

    cfrad.20080604_002217.nc

has the following properties:

  • year: 2008
  • month: 06
  • day: 04
  • hour: 00
  • min: 22
  • sec: 17

DORADE naming convention

The standard DORADE naming convention is: 

    swp.yyymmddhhmmss.radarname.millisecs.fixedAngle_scantype_vnum

where yyy = year - 1900

For example, the file: 

    swp.1080604001951.SPOL.238.4.7_SUR_v036

has the following properties:

  • year: 2008
  • month: 06
  • day: 04
  • hour: 00
  • min: 19
  • sec: 51
  • radar name: SPOL
  • millisec: 238
  • elevation angle: 4.7
  • scan strategy: SUR (surveillance)
  • volume number: 036

The scan strategy can take the following values:

  • SUR: surveillance scan
  • PPI: sector scan
  • RHI: RHI scan
  • IDL: idle
  • SUN: sun scan

UF data naming convention

Universal Format files should have the following naming convention: 

    *yyyymmdd_hhmmss*.uf
where * refers to any character string.

For example, the file: 

    spol_20080604_002217_v064.uf

has the following properties:

  • year: 2008
  • month: 06
  • day: 04
  • hour: 00
  • min: 22
  • sec: 17

FORAY netCDF naming convention

The standard FORAY netCDF naming convention is: 

    ncswp_radar_yyymmdd_hhmmss.msecs_vnum_snum_fixedAngle_scan_.nc

For example, the file: 

    ncswp_SPOL_20080605_235100.123_v054_s007_009.1_SUR_.nc

has the following properties:

  • radar name: SPOL
  • year: 2008
  • month: 06
  • day: 05
  • hour: 23
  • min: 51
  • sec: 00
  • millisec: 123
  • volume number: 054
  • sweep number: 007
  • elevation angle: 9.1
  • scan strategy: SUR (surveillance)

The scan strategy can take the following values:

  • SUR: surveillance scan
  • PPI: sector scan
  • RHI: RHI scan
  • IDL: idle
  • SUN: sun scan
